# VenturoERP 專案簡報指令

> 用於新 AI 對話串的標準開場指令
> 最後更新：2025-01-21

---

## 🎯 給新 AI 對話的統一指令

請複製以下內容給新的 AI 對話串：

```
這是 VenturoERP 2.0 專案，請先閱讀以下資訊：

## 專案位置
/Users/williamchien/Desktop/Venturo/venturoerp-tailwind/

## 必讀文件（依優先順序）
1. PROJECT_CONCEPT_MERGED.md - 專案概念與目前進度
2. ISSUES_TRACKING.md - 問題追蹤與修正記錄
3. DEVELOPMENT_PRINCIPLES.md - 開發原則（必須遵守）
4. LAYOUT_ALIGNMENT_GUIDE.md - 版面對齊規範（強制執行）

## 設計規範
1. 不使用 alert() - 改用 Toast 或 Dialog
2. 不使用 Emoji - 所有圖示用 SVG 或文字
3. 使用 Catalyst UI 元件庫
4. 嚴格遵守版面對齊規範
5. 使用 TypeScript + Tailwind CSS
6. 資料庫使用 Supabase

## 回答規則
1. 先確認理解需求，等待確認後再實作
2. 使用文字排版展示 UI 設計
3. 簡潔回答，不廢話
4. 有優化建議先提出，等待回覆
5. 修改程式碼前先說明改動內容

## 目前狀態
- Phase 1 MVP 已完成
- 雙模式（生活/工作）已實作
- 待辦系統正常運作
- 準備開發 Phase 2 特色功能

請先確認已了解以上資訊，然後告訴我你要協助什麼。
```

## 📝 內部工作規範

### 1. 檔案修改流程
```
1. 先讀取現有檔案
2. 說明要修改什麼
3. 等待確認
4. 執行修改
5. 更新 ISSUES_TRACKING.md
```

### 2. UI 設計流程
```
1. 用文字排版展示設計
2. 確認設計方向
3. 實作程式碼
4. 確保對齊規範
```

### 3. 問題處理流程
```
1. 記錄在 ISSUES_TRACKING.md
2. 分析問題原因
3. 提出解決方案
4. 實作並測試
5. 更新狀態
```

### 4. 進度更新流程
```
1. 完成功能後更新 PROJECT_CONCEPT_MERGED.md
2. 記錄完成日期
3. 標記下一步工作
```

---

## 🚫 絕對禁止

1. **不要使用 Emoji** - 包含文件、程式碼、註解
2. **不要使用 alert()** - 影響使用者體驗
3. **不要忽視對齊規範** - PR 會被拒絕
4. **不要直接修改未確認** - 先討論再行動
5. **不要使用暫時方案** - 一次做對
